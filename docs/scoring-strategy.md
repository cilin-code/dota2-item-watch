# Steam Dota 2 饰品评分推荐策略文档

> 本文档描述「饰品监测」系统的评分与推荐购买策略的完整逻辑。
> 目标不是预测未来价格，而是通过多维数据分析，筛选出当前更适合买入的 Dota 2 饰品。

> 维护说明：本文保留了早期完整策略设计稿中的部分扩展维度，实际运行逻辑以 `backend/engine.py`、`backend/price_semantics.py`、`backend/update_policy.py` 和 README 的说明为准。当前实现采用更保守、更轻量的评分模型，并使用 S/A/B/C/D/E 六档等级。

## 1. 系统概述

### 1.1 设计理念

本系统采用「自底向上」的分析策略：从原始价格数据出发，逐步提取统计特征、趋势特征、技术指标和市场结构信息，最终汇聚成一个 0~150+ 的综合评分。评分不设上限，允许超出 100 分，以便在同等优秀的饰品之间做更精细的区分。

### 1.2 核心原则

- **低价买入是唯一目标**：系统只推荐买入时机，不推荐卖出。
- **多维度交叉验证**：单一指标不足以做出判断，必须多个维度同时指向「低估」。
- **税率不可忽略**：Steam 市场卖出时扣除 13% 交易税，买入价必须足够低才能覆盖这一成本。
- **流动性分级对待**：Dota 2 饰品整体流动性偏低，不能简单排除低交易量饰品，但需更保守评估。

## 2. 数据基础

### 2.1 数据来源

| 数据项 | 来源 | 说明 |
|--------|------|------|
| 当前售价 | Steam /priceoverview API | CNY 计价，currency=23 |
| 历史成交价 | Steam listing 页面嵌入 JS | 从 JPY 按实时汇率转换为 CNY |
| 24h 成交量 | Steam /priceoverview API | 最近 24 小时成交件数 |
| 饰品元信息 | Steam /search/render API | 名称、图标、稀有度等 |

当前售价以 Steam `/priceoverview` quote 为准。listing 订单簿用于卖压、盘口差异和异常提示，不覆盖现价。历史走势、日汇总、周期分析只使用历史成交数据。

### 2.2 历史数据窗口

- **评分计算**：默认使用最近 **90 天** 的历史价格数据。
- **每日汇总**：为性能优化，从 price_snapshots 表按日聚合为 daily_summary。
- **详细视图**：饰品详情页可展示 180 天历史。

### 2.3 数据清洗

使用 **IQR（四分位距）** 方法清洗异常值：

`
Q1 = 第 25 百分位数
Q3 = 第 75 百分位数
IQR = Q3 - Q1
有效 IQR = max(IQR, mean × 0.05)
下界 = Q1 - 1.5 × 有效IQR
上界 = Q3 + 1.5 × 有效IQR
`

保留区间内的价格作为干净数据。若清洗后数据为空，则退回使用原始数据。

## 3. 评分流程

`
原始价格数据
    │
    ▼
┌──────────────┐
│  数据清洗     │ ◄── IQR 去异常值
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  前置过滤     │ ◄── 样本数 < 5、无成交量、CV 超标等情况直接拒绝
└──────┬───────┘
       │ (通过)
       ▼
┌──────────────┐
│  特征计算     │ ◄── 趋势、波动率、分位数、均线、布林带、卖压、智能参考价
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  多维评分     │ ◄── 9 个维度独立打分，然后加总
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  推荐等级     │ ◄── 根据总分映射到 5 个推荐等级
└──────────────┘
`

## 4. 维度 A：价格位置

### 4.1 核心思想

判断当前价格在历史价格分布中处于什么位置。**分位数越低，说明当前价格越便宜**，买入价值越高。

### 4.2 计算方法

`
历史分位 = 低于等于当前价的历史样本数 / 总样本数
近期分位 = (当前价 - 近14天最低价) / (近14天最高价 - 近14天最低价)
`

### 4.3 评分规则（基础分 0 ~ 35）

| 历史分位 | 税后可行 | 税后不可行 | 说明 |
|----------|----------|------------|------|
| ≤ 10% | +35 | +20 | 历史极低位 |
| ≤ 20% | +25 | +14 | 深度低估 |
| ≤ 35% | +14 | +7 | 偏低区间 |
| ≤ 50% | +5 | +2 | 合理偏低 |
| > 65% | -10 | -10 | 价格偏高 |
| > 80% | -25 | -25 | 历史高位，风险大 |

近期分位（近 14 天）加成：
- ≤ 15%：+8 分（近期低位确认）
- > 75%：-6 分（近期追高风险）

### 4.4 设计意图

- 价格处于历史底部区域意味着**下跌空间有限**、**反弹潜力大**。
- 结合税率判断：即使价格低，但如果税后无盈利空间，加分也会打折。
- 近期分位用于确认「低价是持续的」而非「偶然跳水」。

## 5. 维度 B：均线偏离

### 5.1 核心思想

均线代表了市场在一段时间内的「平均共识价格」。**当前价低于均线，说明相对便宜**；低于均线越多，低估程度越大。

### 5.2 计算方法

使用 EMA（指数移动平均）计算 7 日、30 日、60 日、90 日均线。

`
偏离度 = (当前价 - 均线) / 均线 × 100%
`

### 5.3 评分规则（基础分 0 ~ 18）

**30 日均线偏离：**

| 偏离度 | 分数 | 说明 |
|--------|------|------|
| ≤ -10% | +18 | 超跌，严重低于均线 |
| ≤ -5% | +12 | 明显低于均线 |
| ≤ 0% | +6 | 略低于均线 |
| > 15% | -10 | 追高风险 |

接近 30 日最低价（≤ 最低价 × 1.03）：额外 +6 分。

**中长期均线偏离（60 日）：**

| 偏离度 | 分数 | 说明 |
|--------|------|------|
| ≤ -8% | +10 | 中长期低估 |
| ≤ -3% | +5 | 中期偏低 |

**长期均线偏离（90 日）：**

| 偏离度 | 分数 | 说明 |
|--------|------|------|
| ≤ -8% | +6 | 长期低估 |

### 5.4 设计意图

- 短期均线（30 日）权重最大，反映近期市场情绪。
- 中长期均线（60/90 日）权重较小，但多周期同时低估时，信号更强。
- 如果当前价同时低于 30 日、60 日、90 日均线，形成「三重低估」，这是极强信号。

## 6. 维度 C：趋势确认

### 6.1 核心思想

不是所有低价都值得买入。如果价格在持续下跌（趋势向下），抄底可能抄在半山腰。需要确认价格是否已经**企稳或反转**。

### 6.2 计算方法

对最近 7 天的日均价做线性回归：

`
slope = 回归直线的斜率
R² = 回归的拟合优度（0~1，越接近 1 说明趋势越明确）
`

走势状态分类：
- R² > 0.6 且 slope > 0 → **RISING**（上涨）
- R² > 0.6 且 slope ≤ 0 → **FALLING**（下跌）
- R² ≤ 0.6 → **STABLE**（横盘）

### 6.3 评分规则（基础分 0 ~ 14）

| 条件 | 分数 | 说明 |
|------|------|------|
| R² > 0.8 且 slope < 0 | +14 | 下跌趋势高度可预测 → 等反弹 |
| R² > 0.6 且 slope < 0 | +9 | 持续下跌中 |
| R² 适中，负斜率 | +0 ~ +6 | 视 R² 强度递进加分 |
| R² 低（趋势不明） | -2 ~ -5 | 趋势混乱 |

### 6.4 设计意图

- 这里的逻辑是**反直觉**的：当前在下跌但趋势明确的饰品，反而得分更高——因为这意味着下跌已接近尾声，反弹的概率在增大。
- 同时用长期（60 天）斜率做对照，确认这不是短期波动而是中期趋势。

## 7. 维度 D：短期动量

### 7.1 核心思想

最近 3 天的价格变化，反映的是**最即时的市场情绪**。如果最近 3 天跌幅较大，可能意味着恐慌性抛售，是不错的买入时机。

### 7.2 评分规则（基础分 0 ~ 12）

| 3 日涨跌幅 | 分数 | 说明 |
|------------|------|------|
| -10% ~ -3% | +12 | 短期超跌，反弹概率高 |
| -3% ~ 0% | +8 | 小幅回调 |
| 0% ~ +2% | +4 | 温和上涨 |
| > +8% | -3 | 短线涨幅过大 |

### 7.3 设计意图

- 最理想的买入时机：短期微跌（-3%~0%），说明抛压释放但未崩盘。
- 短期暴跌（-10% 以下）：是更激进的买入信号，反弹空间更大。
- 短期大幅上涨：追入风险高，轻微扣分。

---

## 8. 维度 E：走势状态

### 8.1 评分规则（基础分 -12 ~ +12）

| 状态 | 条件 | 分数 | 说明 |
|------|------|------|------|
| STABLE | 横盘 | +10 | 横盘企稳，突破前兆 |
| RISING | 低位启动 | +8 | 趋势确立，上升初期 |
| RISING | 小幅上涨 | +3 | 温和上升 |
| RISING | 涨幅已大 | -3 | 追高风险 |
| FALLING | 加速下跌 (3日 < -8%) | -12 | 仍在暴跌中，观望 |
| FALLING | 持续阴跌 | -5 | 慢刀子割肉 |

### 8.2 设计意图

- **横盘（STABLE）** 是最理想的买入前状态：价格稳定，波动小，一旦突破往往有较好收益。
- 上涨趋势中的饰品，只有在「低位启动」时才加分，高位追涨不仅不加分还会扣分。
- 下跌趋势需要区分「加速暴跌」（恐慌中）和「缓慢阴跌」（钝刀子），前者扣分更重。

## 9. 维度 F：成交量

### 9.1 核心思想

成交量代表了市场的**流动性和活跃度**。高成交量的饰品更容易买入和卖出，滑点小，流动性风险低。

### 9.2 评分规则（基础分 -6 ~ +12）

| 24h 成交量 | 分数 | 说明 |
|------------|------|------|
| ≥ 50 件 | +12 | 高流动性 |
| ≥ 20 件 | +8 | 中等流动性 |
| ≥ 5 件 | +3 | 基本流动性 |
| < 3 件 | -6 | 成交量极低 |

### 9.3 设计意图

- Dota 2 饰品整体成交量偏低，阈值设置比 CS2 等热门游戏宽松很多。
- 成交量极低不直接拒绝，而是扣分 + 标注风险，让用户自行判断。

## 10. 维度 G：风险惩罚

### 10.1 波动率风险

| CV（变异系数） | 扣分 | 说明 |
|----------------|------|------|
| > 0.25 | -22 | 波动率极高，价格不稳定 |
| > 0.15 | -12 | 波动率偏高 |
| > 0.08 | -4 | 轻微波动 |

### 10.2 卖压风险

| 卖压指标 | 扣分 | 说明 |
|----------|------|------|
| > 2.5 | -14 | 卖单堆积严重 |
| > 1.2 | -6 | 卖单偏多 |

### 10.3 流动性风险

| 条件 | 扣分 | 说明 |
|------|------|------|
| 冷门饰品 (THIN) | -5 | 交易稀薄 |
| 被前置过滤拒绝 | -25 | 硬伤 |

### 10.4 设计意图

- 这些惩罚项的作用是**阻止系统在高风险情况下给出乐观建议**。
- 即使价格位置很好，如果波动率高、卖压大，也不应该推荐。

## 11. 维度 H：多信号共振

### 11.1 核心思想

单一指标可能误判，但**多个独立指标同时指向「低估」** 时，信号的可靠性大幅提升。这是系统中最关键的加分机制。

### 11.2 共振信号清单

系统检测以下 7 个信号：

| 序号 | 信号条件 | 含义 |
|------|----------|------|
| 1 | 历史分位 ≤ 30% | 价格处于历史底部 |
| 2 | 当前价 ≤ 30日均线 × 0.95 | 价格低于均线 5% 以上 |
| 3 | 7 日涨跌幅 < 0 | 短期在跌 |
| 4 | 24h 成交量 ≥ 10 件 | 流动性充足 |
| 5 | 走势状态为 STABLE 或 RISING | 企稳或上升 |
| 6 | 当前价 ≤ 60日均线 × 0.95 | 中长期低估 |
| 7 | 当前价 ≤ 90日均线 × 0.95 | 长期低估 |

### 11.3 评分规则（基础分 0 ~ 25）

| 共振信号数 | 分数 | 说明 |
|------------|------|------|
| 7 个 | +25 | 多周期全面共振——最强信号 |
| 6 个 | +18 | 多周期强烈共振 |
| 5 个 | +25 | 五重信号共振——强烈买入 |
| 4 个 | +15 | 四重信号共振——高确定性 |
| 3 个 | +8 | 多重信号共振 |
| ≤ 2 个 | 0 | 信号不足 |

### 11.4 设计意图

- 这就是为什么某些饰品能突破 100 分：共振加分可以叠加在其他维度之上。
- 7 个信号全部满足意味着：所有时间维度（短期、中期、长期）、所有指标类型（价格位置、均线、趋势、成交量）都指向同一个结论。
- 5 重共振的加分（+25）与 7 重相同，是因为 5 重已经是极强信号，额外 2 重不会线性增强。

## 12. 维度 I：税率调整（分段计算）

### 12.1 核心思想

Steam 市场卖出时收取 **13% 交易税**。这意味着：

`
买入价 ¥100 → 卖出需 ≥ ¥114.94 才能回本（100 / 0.87 ≈ 114.94）
`

系统计算的盈亏平衡倍数是：

`
`
# 分段计算保本倍数
如果 当前价 >= 1.60：
    保本倍数 = 1 / (1 - 0.13) ≈ 1.149
如果 当前价 < 1.60：
    最低总费用 = 1.60 × 0.13 = 0.208
    实际到手 = 当前价 - 0.208
    保本倍数 = 当前价 / 实际到手
`
`

### 12.2 分段保本倍数表

| 当前售价 | 实际税费 | 实际税率 | 保本倍数 | 需涨幅 |
|----------|----------|----------|----------|--------|
| ¥0.30 | ¥0.208 | 69.3% | 3.26 | +226% |
| ¥0.50 | ¥0.208 | 41.6% | 1.71 | +71% |
| ¥0.80 | ¥0.208 | 26.0% | 1.35 | +35% |
| ¥1.00 | ¥0.208 | 20.8% | 1.26 | +26% |
| ¥1.50 | ¥0.208 | 13.9% | 1.16 | +16% |
| ¥1.60+ | 13% | 13.0% | 1.149 | +14.9% |

### 12.3 计算方法

`
保本卖出价 = 当前价 × 保本倍数
30日均价盈利率 = (30日均价 - 保本卖出价) / 保本卖出价 × 100%
90日均价盈利率 = (90日均价 - 保本卖出价) / 保本卖出价 × 100%
`

### 12.3 评分规则（基础分 -22 ~ +20）

| 30日均价盈利率 | 分数 | 说明 |
|----------------|------|------|
| > 5% | +14 | 均价已覆盖税率，税后仍有可观空间 |
| > -3% | +6 | 接近保本线，少量涨幅即可盈利 |
| > -8% | -4 | 需小幅上涨才覆盖税率 |
| > -15% | -12 | 税率侵蚀严重 |
| ≤ -15% | -22 | 几乎无盈利可能 |

**90 日均线加成：**
- 90日均价盈利率 > -5%：+6 分

### 12.4 对价格位置评分的联动

A 维度（价格位置）的加分在 	ax_viable 为 False 时会打折（约打 6 折），因为即使价格位置低，税后无利可图也不值得买入。

### 12.5 设计意图

- 这是本系统最重要的现实考量：**Steam 的 13% 税率是一个无法回避的成本**。
- 即使当前价格处于历史最低的 5% 分位，如果均价低于保本线，仍然不推荐。
- 建议在制定实际交易策略时，将保本卖出价（当前价 × 1.149）作为必须达到的最低卖出目标。

## 13. 推荐等级划分

### 13.1 六档等级体系

| 总分范围 | 等级 | 含义 |
|----------|------|------|
| ≥ 120 | S | 极强信号 |
| 100 ~ 119 | A | 强信号 |
| 85 ~ 99 | B | 较好信号 |
| 70 ~ 84 | C | 可观察信号 |
| 55 ~ 69 | D | 信号偏弱 |
| < 55 | E | 风险或数据不足 |

### 13.2 最低分保障

`
最终分数 = max(0, 四舍五入(总分))
`

确保分数不会低于 0。

## 13.3 更新准入策略

评分低不等于删除饰品，而是降低批量更新频率。手动批量更新时，系统会根据 `update_after` 判断低分饰品是否进入本轮更新：

| 分数 | 冷却时间 |
|------|----------|
| >= 20 | 不限制 |
| 15 ~ 19.9 | 12 小时 |
| 10 ~ 14.9 | 24 小时 |
| 5 ~ 9.9 | 72 小时 |
| < 5 | 168 小时 |

以下情况绕过冷却：Steam 热门命中、收藏、有买入记录、有启用预警、用户指定刷新。

## 14. 前置过滤规则

在进入评分之前，系统会先进行多道过滤。被过滤掉的饰品仍会得到一个基础评分，但整体分数会偏低（有 -25 的拒绝惩罚）。

### 14.1 过滤规则表

| 过滤条件 | 拒绝原因 | 备注 |
|----------|----------|------|
| 干净价格样本 < 5 | 历史样本不足 | 硬拒绝 |
| 24h 成交量 = 0 | 无近期成交量 | 硬拒绝 |
| 趋势持续下跌 (FALLING) | 趋势持续下跌 | 硬拒绝 |
| 趋势信号不足 (UNKNOWN) | 趋势信号不足 | 硬拒绝 |
| 横盘但斜率 < -0.005 | 近期偏弱 | 仅 STABLE 状态 |
| 冷门品且斜率为负 | 冷门品近期偏弱 | 仅 THIN |
| CV > 自适应阈值 | 价格波动过大 | 阈值因价格区间而异 |
| 历史分位 > 位置上限 | 处于历史高位 | 上限因状态而异 |
| 近期分位 > 位置上限 | 处于近期高位 | 上限因状态而异 |
| 价格 > 布林上轨 | 突破布林上轨 | 技术面过强 |
| 卖压 > 阈值 (LIQUID: 2.0, THIN: 1.5) | 卖单压力过大 | 盘口结构不利 |
| 无法计算智能参考价 | 无法计算参考价 | 数据不足 |

### 14.2 波动率自适应阈值

| 价格区间 | CV 阈值 | 说明 |
|----------|---------|------|
| ≤ ¥15 | 0.08 | 低价品允许更大波动 |
| ¥15 ~ ¥100 | 线性从 0.08 到 0.04 | 逐渐收紧 |
| ≥ ¥100 | 0.04 | 高价品要求更稳定 |

冷门品 (THIN)：阈值再 × 0.8（更严格）。
